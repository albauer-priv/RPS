# ADR-015: Readiness Visualization

**Status:** Accepted  
**Date:** 2026-02-01  

## Context

Users need clear signals for missing/stale/blocked steps before planning runs.

## Decision

Readiness uses status badges with reasons and fix CTAs. Missing/blocked items are expanded by default.

## Consequences

- Clear prioritization of issues.
- Consistent UX across planning flows.

## Exceptions

None.
